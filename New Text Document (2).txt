```
POST   /api/v1/halaqah/:id/duas         → add dua request (stored anonymous)
POST   /api/v1/halaqah/:id/duas/:duaId/pray → mark "I made dua for this"
DELETE /api/v1/halaqah/:id/duas/:duaId  → delete own dua (author only)

# Naseeha (anonymous encouragement)
POST   /api/v1/halaqah/:id/naseeha      → send anonymous encouragement
GET    /api/v1/halaqah/:id/naseeha      → my received encouragements
```

### Screen Time

```
POST   /api/v1/screentime/log           → log today's phone usage
GET    /api/v1/screentime/today         → today's usage + opportunity cost
GET    /api/v1/screentime/week          → weekly trend
PUT    /api/v1/screentime/budget        → set daily phone budget
```

## SCORE COMPUTATION SERVICE

### `backend/src/modules/score/score.service.ts`

```typescript
// Service that runs daily score computation
//
// computeDailyScore(uid: string, date: string): Promise<DailyScore>
//   1. Fetch today's prayer log → compute salah score
//   2. Fetch today's quran sessions → compute quran score
//   3. Fetch today's adhkar log → compute adhkar score
//   4. Fetch today's completed tasks → compute productivity score
//   5. Fetch lifestyle log (sleep, phone, water) → compute health score
//   6. Fetch bonus activities (fasting, sadaqah, family, muhasaba) → compute bonus
//   7. Sum all, cap at 100
//   8. Save to users/{uid}/daily_scores/{date}
//   9. Update shared_scores for any halaqah memberships
//
// computeWeeklyRing(uid: string): Promise<number>
//   Average of last 7 daily scores as percentage
//
// computeRank(uid: string): Promise<RankTitle>
//   Based on 7-day rolling average
//
// computeStreak(uid: string): Promise<number>
//   Count consecutive days with score >= 40
//
// updateSharedScores(uid: string): Promise<void>
//   Update users/{uid}/shared_scores with ONLY rank, ring, streak
//   NEVER include breakdown or worship details
//
// IMPORTANT: Score computation happens:
//   - On demand when user opens dashboard
//   - Automatically at end of day (11:59 PM user's timezone)
//   - When user logs any activity (prayer, quran, task)
//   - Cached for 5 minutes to avoid excessive computation
```

## CHALLENGE SYSTEM

### AI-Generated Weekly Challenges

```typescript
// POST /api/v1/ai/generate-challenge
// 
// The AI analyzes the Halaqah's aggregate performance:
//   - If most members have low adhkar scores → suggest adhkar challenge
//   - If most members have low quran scores → suggest quran challenge
//   - If it's Monday/Thursday → suggest fasting challenge
//   - If Ramadan is approaching → suggest preparation challenge
//   - If a member (anonymous) is struggling → suggest easy challenge
//
// Challenge types:
//   "Complete morning adhkar every day"
//   "Read 1 juz of Quran this week"
//   "Fast Monday and Thursday"
//   "Pray all 5 prayers on time for 3 consecutive days"
//   "Do 100 istighfar daily"
//   "Make dua for someone each day"
//   "Contact a family member you haven't spoken to"
//
// RULES:
//   - Challenges are SHARED — same for all members
//   - Progress is shown as GROUP aggregate (3/5 today)
//   - Individual completion is PRIVATE
//   - Challenges last 1 week
//   - New challenge auto-generated every week
//   - Creator can override with custom challenge
```

## NASEEHA SYSTEM (Anonymous Encouragement)

```typescript
// How it works:
//
// 1. System detects a member's streak broke (score < 40 for 3+ days)
// 2. System sends notification to Halaqah: 
//    "A brother in your Halaqah needs encouragement"
//    NO name. NO details. Just the prompt.
// 3. Members can tap "Send Naseeha" to write an anonymous message:
//    - Pre-written options: "Keep going!", "Allah sees your effort", 
//      "Even small deeds are beloved to Allah", "We're all human"
//    - Or custom message (moderated — no identifying info allowed)
// 4. The struggling member receives: "A brother sent you naseeha: ..."
//    They never know who sent it.
//
// Why anonymous:
//   - Removes ego from both sides
//   - The sender can't boast about giving advice
//   - The receiver doesn't feel judged by a specific person
//   - Mirrors the Sunnah of private naseeha
//
// IMPORTANT: This feature can be disabled per-Halaqah in settings
// Some people prefer not to be monitored even anonymously
```

## FIRESTORE SECURITY RULES (Halaqah-specific)

```javascript
// Halaqah rules
match /halaqat/{halaqahId} {
  // Only members can read halaqah data
  allow read: if request.auth.uid in resource.data.members;
  // Only creator can update settings or delete
  allow update: if request.auth.uid == resource.data.createdBy;
  allow delete: if request.auth.uid == resource.data.createdBy;
  // Anyone authenticated can create
  allow create: if request.auth != null;
  
  // Dua wall — members can read, anyone can create (if member)
  match /dua_wall/{duaId} {
    allow read: if request.auth.uid in get(/databases/$(database)/documents/halaqat/$(halaqahId)).data.members;
    allow create: if request.auth.uid in get(/databases/$(database)/documents/halaqat/$(halaqahId)).data.members;
    // Only author can delete their own dua
    allow delete: if request.auth.uid == resource.data.authorUid;
  }
  
  // Messages — same rules
  match /messages/{msgId} {
    allow read: if request.auth.uid in get(/databases/$(database)/documents/halaqat/$(halaqahId)).data.members;
    allow create: if request.auth.uid in get(/databases/$(database)/documents/halaqat/$(halaqahId)).data.members;
  }
}

// Shared scores — CRITICAL privacy rules
match /users/{uid}/shared_scores/{weekId} {
  // Owner can read/write their own
  allow read, write: if request.auth.uid == uid;
  // Halaqah members can read (validated in backend, not here for perf)
  // Backend handles the privacy filter — never returns worship details
}

// Daily scores — PRIVATE, only owner
match /users/{uid}/daily_scores/{date} {
  allow read, write: if request.auth.uid == uid;
  // NEVER allow other users to read this
}
```

## EXECUTION RULES (Sibaq-specific)

```
1.  shared_scores MUST NEVER contain worship details — this is a PRIVACY requirement
2.  Dua wall MUST be anonymous — authorUid stored but NEVER returned in API responses
3.  Naseeha MUST be anonymous — fromUid stored but NEVER exposed
4.  Streak-break notification MUST NOT include the person's name
5.  Leaderboard sorts by weeklyRingPercent, NOT exact score
6.  Challenge completion shows group aggregate, NEVER individual
7.  Score breakdown (which prayers, how many pages) is NEVER shared
8.  When a user leaves a Halaqah, ALL their shared data is deleted
9.  Max 7 members per Halaqah — keeps it intimate and meaningful
10. Score threshold for streak (40) is intentionally LOW — encourage consistency over perfection
11. The AI coach NEVER mentions a user's score to other users
12. Phone usage penalty is WITHHOLDING points, never NEGATIVE scoring
```
